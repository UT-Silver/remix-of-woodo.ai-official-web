公式与校准:每个系数的来历、证据与边界

本页把 final test time 公式拆到最后一颗螺丝:每个参数是什么、系数怎么来的(校准方程逐步展示)、有什么文献/渠道证据、哪里是薄弱环节。诚实原则:凡是"归因"而非"测量"的地方都明说。

A · 公式总览

GPU / ASIC: T_FT = [ 2.75·X/E·(1+0.54·(D−1)) + 20·Hs·(Hl/8)·bus ]·κ + 40(秒)
ATE 总机时: T_ATE = T_FT × (1 + γ + 0.4·cw),γ:GPU=0.8 / ASIC=0.6
CPU: T_FT = 1.5·X/E + 30;T_ATE = 2×T_FT(SLT 分钟另账,才是 CPU 主战场)
Networking: T_FT = lanes·t_lane·rateF + 0.8·X/E + 40(loopback 主导,非 scan)
CPO: T_光学 = 引擎数×单PIC秒 + 电学秒(光学设备账,不进 ATE 桥)
衰减层(需求桥): T_eff(年) = T_FT × (1−d)^(年−量产年),d 默认 7.5%/yr(你的表假设)
口径声明:T_FT 是"投产期单颗 final test 口径(含轮次)"——你的三锚点(360/1080/1512)即此口径。它不含 burn-in(小时级另账)、不含 SLT(分钟级另账)、不含 CP/die-sort(通过 T_ATE 的乘数进入)。Advantest CEO 的"3-4×Hopper"是全流程数十次插入的合计口径,不能与单次 FT 直接对表——这是最常见的口径混淆。

参数/系数含义取值与来历证据与警告
+40(T_C)DC 参数/连续性/供电测试等与规模无关的固定项假设值;A100~Rubin 通吃行业常识级;在小芯片(TPU v4/v5e)占比高,大芯片可忽略

λ=0.54(多 die 系数,×(1+λ·(D−1)))跨 reticle-die 封装的测试超线性:die 间互连测试(NV-HBI 10TB/s)、KGD 贴装后复测、多 die 同步、热约束降并测由你的 B200=1080s 锚点反解:1080=2.75·208·(1+λ)+160+40 → λ=0.538机制有文献:IEEE 1838-2019(多插入标准)、Taouil ATS'10/3D-COSTAR(KGD+复测成本最优)、2.5D interposer 测试调度、SemiEng 热致降并测;2026补强:Advantest IR技术说明会2024-11(官方)确认multi-die组装"增加FT测试内容"并讨论新增die-level test插入,IEEE 1838仍为现行标准(修正案P1838a扩展中)。系数无文献——你的 1080 锚点是 λ 的唯一定量来源。旁证:大摩 2024-12 自己给过"Blackwell=3×Hopper"倍率,3×360=1080 精确自洽;Data Gravity 独立口径">20 分钟"。D 的判定:B200/Rubin=2(NV-HBI 类互连);MI300X=1(SoIC 3.5D 由 cw·δ 吸收);MI250X=1(EFB 双 GCD 软件上是两颗独立 GPU,无跨 die 链路测试)
20·Hs·(Hl/8)·bus(HBM 接口项)每颗 HBM 栈的接口/训练/repair 相关 FT 时间h=20s/栈(8-Hi、1024-bit 基准);层数线性(Hl/8);HBM4 2048-bit → bus=2Advantest 官方:层数 8→12→16 直接拉长测试时间、堆叠后测试机台数多于常规 DRAM;绝对秒数(20)是校准残差分配,无独立文献。注意这是 logic 侧 FT 里的 HBM 接口测试,不是 HBM 厂自己的 KGSD 测试(那在 memory tester 账上)
κ(内容/成熟度系数)新增测试内容(新精度格式/加严)与多温轮次的净乘数默认 1.0;GB300=1.2(增强 die+加严);Rubin Ultra=0.95GB300 案例:工商时报"测试时间已恢复至 1000 秒"证明测试时长是动态管理的——先优化压下、良率/品控事件后加回
γ + 0.4·cw(T_ATE 乘数)把单次 FT 换算成全流程 ATE 机时:CP(晶圆级)+ die-sort(CoWoS-L 的 KGD 闸门)+ retestγ:GPU=0.8 / ASIC=0.6(ASIC 插入结构更简);cw=1 时 +0.4(CoWoS-L/SoIC 的 die-sort)归因自 Advantest CEO"数十次插入"与 IEEE 1838 多插入框架;比例为假设,无文献系数。需求桥用的是 T_ATE 而非 T_FT
burn-in / SLT小时级/分钟级独立参数,不进 T_FTRubin burn-in 3h→5-6h(大摩✓);TPU ~4h;SLT 分钟级假设三种设备三本账:FT=V93000、burn-in=烧机炉(京元电自制)、SLT=Chroma/鸿劲系统。公开佐证(DCD 2024):KYEC为Blackwell烧机炉600W→1kW、流程FT→burn-in→FT→SLT;JEDEC JESD22-A108G(2022)明文burn-in用于筛infant mortality。Weibull 早夭筛选理论支撑 burn-in 存在,但小时数弹性极大(HVST 可压缩 90%),AI 趋势反向(burn-in 回归+SLT 全检化)

C · 校准方程组(三锚点逐步求解)

锚点1 · H100=360s(X=80,D=1,Hs=5,Hl=8,bus=1,κ=1,E=1):
  360 = k·80 + 20·5 + 40 → k = 220/80 = 2.75
锚点2 · B200=1080s(X=208,D=2,Hs=8):
  1080 = 2.75·208·(1+λ) + 160 + 40 → λ = 880/572 − 1 = 0.538 ≈ 0.54
锚点3 · Rubin=1512s(X=336,D=2,Hs=8,Hl=12,bus=2):
  1512 = 2.75·336/E·1.54 + 480 + 40 → E = 1423/992 = 1.43 ≈ 1.45
独立复现(未用于校准的交叉验证):
  A100(X=54):模型 289 vs 你的表 288 ✓  ·  MI455X:模型 1525 vs 你的表 1512 ✓  ·  MI500X:模型 2282 vs 你的表 2268 ✓（注:H20 模型380 vs 表288 差32%,是待渠道复核分歧而非命中,见 E 节）
为什么这套校准可信:3 个自由参数(k、λ、E_Rubin)精确解释 3 个锚点是"拟合",但模型用同一套系数独立命中了 3 个未参与校准的数字(A100/MI455X/MI500X)——这是"模型有物理内容"而非"曲线拟合"的证据。同时它也解释了 B200 的 3.0× 之谜:晶体管贡献 2.6×,λ 多 die 项贡献余下 ~15%(die 间互连测试+KGD 复测+降并测)。

F · 已知局限

① 线性 scan 标度是简化——文献支持亚线性,误差由 E/κ 吸收;② λ=0.54 无文献系数,单锚点校准;③ TPU 全系晶体管数为推算;④ 三锚点本身是渠道口径,无官方披露;⑤ 衰减层与加严(burn-in 回归/SLT 全检)的净效应不确定;⑥ 未建模并测数 S 的代际变化(藏在 ATE 需求桥的校准系数里);⑦ roadmap 级 SKU(Rubin Ultra 之后)所有参数都是占位。模型的正确用法:相对比较与敏感性,而非绝对预测;每个新数据点(尤其你的渠道)都应该回填校准。

ATE 需求桥 · SKU 全联动:TAM = 校准系数 × Σ(出货 × T_ATE)
